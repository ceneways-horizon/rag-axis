## Step 2: Issues to Create (I create these via GitHub MCP)

### Phase 0 — Contract Layer (Days 1–2)

**Issue 1 — `[phase-0] Feature 1+2+3: Core domain types — Provenance, Cost, Failure Taxonomy`**

Implement all frozen dataclasses and the typed failure taxonomy:

- **Feature 1 — Provenance Enforcement Engine**
  - `Chunk(content="test")` with no provenance → raises `MissingProvenanceError` at `__post_init__`
  - Full provenance required: `parent_doc_id`, `position`, `embedding_model_id`
  - `assert_provenance(chunk)` utility function
  - Mutation attempt → `FrozenInstanceError` (enforced by `frozen=True`)

- **Feature 2 — Cost Transparency Layer**
  - `CostReport` is a return type field on every `PipelineResult`, not a log side-effect
  - Fields: `stage_breakdown: dict[str, StageCost]`, `estimated_cost_usd: Decimal`, `cache_hit: bool`
  - `StageCost`: `tokens_consumed: int`, `latency_ms: float`
  - Stages: `embedding`, `retrieval`, `reranking`, `context_assembly`, `generation`
  - Zero cost data in logs — only in return types

- **Feature 3 — Typed Failure Taxonomy**
  - Base hierarchy: `RagAxisError` (fatal) → `DegradedError` (system continues, warning emitted)
  - Named types from `aiprims.rag` + extensions in `rag_axis.core.errors`:
    `EmptyRetrievalError`, `MissingProvenanceError`, `CorpusVersionMismatchError`,
    `StaleIndexError` (degraded), `ScoreCollapseWarning` (degraded)
  - Every error carries: `message: str`, `context: dict`, `run_id: str`
  - Zero bare `Exception` raises anywhere in core or adapters (I4)

- **Feature 6 — Confidence Calibration** (lives on `RetrievalResult`)
  - `score: float | ConfidenceUnknown` — `ConfidenceUnknown` is a sentinel dataclass, not `None`
  - `RankedResult.rerank_score: float | ConfidenceUnknown` same pattern after reranking

- **Feature 7 — Immutable Stage Boundaries**
  - All cross-stage types are `@dataclass(frozen=True)`: `Document`, `Chunk`, `RetrievalResult`,
    `RankedResult`, `ContextWindow`, `PipelineResult`, `CostReport`
  - `dataclasses.replace()` is the only sanctioned "modification" path
  - `PipelineResult` and `RetrievalResult` are Pydantic discriminated unions (ADR-0001 exception for stage-crossing)

Acceptance tests:
```python
Chunk(content="x")  # raises MissingProvenanceError
result.cost.stage_breakdown["retrieval"].tokens_consumed  # int
result.cost.estimated_cost_usd  # Decimal
result.score  # float or ConfidenceUnknown instance
result.chunks[0].parent_doc_id = "x"  # raises FrozenInstanceError
```

**Issue 2 — `[phase-0] Feature 4: Adapters — Corpus Version Locking`**

Define all three adapter Protocols + version locking at init:

- `LLMAdapter`, `EmbedderAdapter`, `VectorStoreAdapter` as `typing.Protocol` with `runtime_checkable` (ADR-0002)
- **Feature 4 — Corpus Version Locking**
  - `CorpusVersion` dataclass: `embedding_model: str`, `version: str`, `created_at: datetime`
  - `VectorStoreAdapter.__init__()` validates: `embedder.model_id == corpus.embedding_model`
  - Mismatch → `CorpusVersionMismatchError` at init, before first query (not mid-query)
  - `EmbedderAdapter` exposes `model_id: str` and `model_version: str`
- Each Protocol carries `model_version`, `schema_version`, `corpus_version`
- Reference no-op implementations for testing (no provider deps in core — I5)

Acceptance test:
```python
store = VectorStoreAdapter(corpus_version=CorpusVersion(embedding_model="text-ada-002", ...))
embedder = MockEmbedder(model_id="text-embedding-3-small")
Pipeline(store=store, embedder=embedder)  # raises CorpusVersionMismatchError
```

### Phase 1 — Observability Primitives (Days 3–5)

**Issue 3 — `[phase-1] rag_axis.ingestion — Loaders and document registry`**

- Lives at `rag_axis/ingestion/` — **top-level package, same level as `rag_axis.core`**, not nested inside it.
- The existing `rag_axis/core/ingestion/` stub directory must be removed and replaced by `rag_axis/ingestion/`.
- File/text loaders, metadata extraction, document registry
- All loaded objects produce `Document` (from `rag_axis.core`)
- I1 provenance set at load time (`source_uri`, `loader_id`, `loaded_at`)
- I4 typed errors for IO failures: `DocumentLoadError`, `UnsupportedFormatError`

**Issue 4 — `[phase-1] rag_axis.chunking — 4 strategies`**

- Fixed-size, semantic, structural, parent-child chunkers
- Every output `Chunk` satisfies I1 (`parent_doc_id`, `position`, `embedding_model_id` = `None` pre-embedding, set by embedder)
- I2: no content dropped silently — if chunker discards content, `ContextTruncationEvent` emitted

**Issue 5 — `[phase-1] rag_axis.embedding — Batch embedding with cost tracking and version lock`**

- Batch embed via `EmbedderAdapter`
- Sets `chunk.embedding_model_id` on every output (completes I1)
- Model version locked at corpus build time — mismatch already caught at adapter init (Feature 4)
- Every call returns `CostReport` with embedding tokens and latency (Feature 2)

---

**Issue 6 — `[phase-1] Feature 5+8: Context Truncation Audit Trail + Relationship Graph (rag_axis.context)`**

- `ContextBudget`: max tokens for context window
- **Feature 5 — Context Truncation Audit Trail**
  - `ContextTruncationEvent` dataclass: `tokens_dropped: int`, `chunks_dropped: int`, `reason: str`, `stage: str`
  - Three truncation policies: `TailTruncation`, `HeadTruncation`, `ScoredTruncation`
  - `ContextAssemblyReport.truncation_events: list[ContextTruncationEvent]`
  - Silent truncation is a contract violation (I2)
- **Feature 8 — Relationship Graph (Chunk Linkage)**
  - Owned entirely by `rag_axis.context`. **Never bleeds into `rag_axis.retrieval`.**
  - Retrieval returns flat `RetrievalResult` objects — no relationship logic, no graph construction.
  - Relationship detection runs inside the context assembler, after retrieval results are handed off.
  - `ChunkRelationship`: `source_chunk_id`, `target_chunk_id`, `type: RelationshipType`, `confidence: float`
  - `RelationshipType` enum: `ADJACENT`, `SHARED_ENTITY`, `SEMANTIC_OVERLAP`, `PARENT_CHILD`
  - `ContextWindow.assembly_report.relationships: list[ChunkRelationship]`
  - Detection: adjacency by position+parent, parent-child from chunking metadata
- Deduplication and ordering strategies in `ContextWindow`

> **SoC boundary**: `rag_axis.retrieval` answers "which chunks are relevant?" and stops. `rag_axis.context` answers "how do these chunks relate?" Relationship graph logic must not be added to any retrieval module.

Acceptance tests:
```python
# budget=100 tokens, chunks=1000 tokens
report.truncation_events[0].tokens_dropped  # int > 0
report.truncation_events[0].stage  # "context_assembly"
report.relationships  # [ChunkRelationship(type=ADJACENT, ...)]
```

### Phase 2 — Contract Enforcement (Days 6–10)

**Issue 7 — `[phase-2] Feature 9: rag_axis.retrieval — Dense + Sparse + Score Collapse Detection`**

- Dense (vector) + sparse (BM25/SPLADE) retrieval via `VectorStoreAdapter`
- Hybrid RRF fusion: `score(d) = Σ 1/(k + rank(d))`, k=60 default (ADR-0003), configurable via `RetrievalConfig`
- Both stream scores exposed in every `RetrievalResult`
- I3: `score: float | ConfidenceUnknown` on all results (Feature 6)
- **Feature 9 — Score Collapse Detection**
  - After retrieval: if `max(scores) - min(scores) < 0.05` → emit `ScoreCollapseWarning`
  - `ScoreCollapseWarning` is `DegradedError` — system continues, ranking is flagged as unreliable
  - Warning carries: `score_range`, `n_results`, `strategy_used`

Acceptance test:
```python
results = retriever.retrieve(query, top_k=10)  # all scores ≈ 0.5
# ScoreCollapseWarning emitted
results[0].score  # ConfidenceUnknown (or float)
```

**Issue 8 — `[phase-2] rag_axis.reranking — Cross-encoder with typed score`**

- Cross-encoder reranking via `LLMAdapter`
- `RankedResult.rerank_score: float | ConfidenceUnknown` (Feature 6 pattern)
- Score normalisation across candidates
- Score collapse check applied post-rerank as well (Feature 9)
- I4: typed errors — `RerankerTimeoutError`, `RerankerUnavailableError`

**Issue 9 — `[phase-2] rag_axis.generation — Prompt assembly and output parsing`**

- Prompt assembly from `ContextWindow`
- LLM call via `LLMAdapter`
- Schema-enforced output parsing: wrong format → `OutputSchemaError` (Feature 3)
- Citation injection from chunk provenance (I1 provenance flows here)
- Every call returns `PipelineResult` with embedded `CostReport` (Feature 2)

### Phase 3 — Production Hardening (Days 11–14)

**Issue 10 — `[phase-3] rag_axis.guards — Input/output guards`**

- Input guards: query validation, length limit, PII detection hook
- Output guards: schema enforcement, confidence threshold
- Fallback strategies: typed, not silent (Feature 3)
- I4: all guard failures typed

**Issue 11 — `[phase-3] Feature 11: rag_axis.cache — Semantic cache with embedding similarity`**

- **Feature 11 — Semantic Cache**
  - Cache stores: `(query_embedding, result, timestamp)`
  - Lookup: embed query → cosine similarity vs. stored embeddings
  - Threshold default: `0.95` (configurable in `CacheConfig`)
  - Hit: skip retrieval+generation, return cached result with `CostReport.cache_hit = True`
  - Exact cache (hash match) checked first; semantic cache second
- `EmbeddingCache`: avoid re-embedding identical document content

Acceptance test:
```python
pipeline.query("what is X?")  # cache miss, stores embedding
pipeline.query("explain X")   # cosine > 0.95 → cache hit
result.cost.cache_hit  # True
```

**Issue 12 — `[phase-3] Feature 12+10: rag_axis.telemetry — OTel Native + Inline Eval Hooks`**

- **Feature 12 — OTel Native (No Opt-In)**
  - Spans emitted for every stage: `pipeline`, `retrieval`, `reranking`, `context_assembly`, `generation`
  - Span attributes: `tokens`, `latency_ms`, `cost_usd`, `strategy`, `cache_hit`
  - No environment variable needed to enable — active by default
  - Opt-out via `TelemetryConfig(enabled=False)` (explicit, not default)
- **Feature 10 — Inline Evaluation Hooks**
  - `EvaluationHook` Protocol: `async def evaluate(result: PipelineResult) -> EvaluationReport`
  - Hooks registered at pipeline init, called per-query, async non-blocking
  - `EvaluationReport` routed to OTel as a span event — not to logs
  - Test: register mock hook → query → hook called with `PipelineResult`

**Issue 13 — `[phase-3] Feature 13: rag_axis.corpus — Stale Index Detection`**

- `CorpusVersion.created_at: datetime` tracked (set at index build)
- Query-time check: `now - corpus.created_at > staleness_threshold` (default 30 days)
- Emit `StaleIndexError` (a `DegradedError`) — system continues, warning logged to OTel span
- Threshold configurable via `CorpusConfig(staleness_threshold_days=30)`

Acceptance test:
```python
corpus = CorpusVersion(created_at=datetime.now() - timedelta(days=40))
pipeline.query("x")  # StaleIndexError emitted (degraded, not fatal)
```
